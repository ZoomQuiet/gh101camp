蟒营®私教

1v1 场景中精确服务

背景

2019年底, 叕被创业后, 主要精力放回团队/开发管理;

蟒营®定期报名/开课/结业/... 滚动式小班服务, 无法保证时间投入, 所以, 暂停, 变成预报名制:

  • 42元预订费
  • 先入群和同好们开始预热
  • 积累到 42+ 人就开班

现象

但是, 互联网注意力实在太稀缺了, 一但没有持续的宣传, 关注和报名一下子就佛系了;

2021年用了7个月, 才有8位学员各种渠道寻找来;

2022目测也差不多节奏;

分析

原先一定要足够人数才开班原因是:

中后段小组团队协同是关键形式,每个小组至少要3~7人,至少要4~8组, 才能形成足够竞争压力逼出大家潜力; 另外, 每次直播要准备不少时间, 只针对少量学员, 现场反应过于零散, 无法快速实现共性, 效能不高.

但是

每个月总是有很久前的学员, 介绍新朋友找来预约想学编程; 实在是编程现在太热, Python 实在太知名, 很多场景中, 好象不会 Python 就要世界未日一般, 而其它网络课程基本上多数是面向:复购率 来设计, 并不关心, 也没资源关心每个具体学员探索过程中具体问题;

所以

蟒营®价值一直在, 只是缺少一个更加个性的提供渠道;

直到一位"大师姐", 复读多次, 每次都因为各种原因没能坚持下来; 特地找到大妈, 发愿: "这次无论如何都要完成自己的作品", 一商量, 发现其实最需要的, 并不是常见知识讲解, 或是代码演示, 而是更加直入内心的调试陪练:

  • 在各种场景中实时给予对应提醒:
    • 探索互联网时, 关键词迭代过程
    • 官方文档通读时, 如何快速定位
    • 开始项目构建时, 如何规划目录
    • 进入代码调试时, 具体问题如何着手思考
    • ...
  • 也就是说:
    • 以学员私人项目为核心
    • 以学员自行开发为线索
    • 以学员编程习惯积累为目标
    • ...定期直播配合持续项目工程指引, 在开发中学习开发, 理解 Python , 养成 Pythonic 习惯

具体过程:

  • 定期进行直播陪练:
    • 具体时间具体商议, 比如:
    • 每周5 2100~2200 项目迭代直播, 演示进展, 规划下周目标
    • 每周2 2100~2200 半周QA, 针对上周任务推进答疑, 直播调试
  • 收费:
    • 按照 342元/小时 收费
    • 每次具体服务多久, 具体核算
  • 提供专用 gitlab 工程仓库
    • 以 Issue 驱动
    • 以 代码仓库 为核心输出渠道
    • 以 wiki 为私人知识仓库
    • 以 腾讯会议为直播界面
  • 一般6~12周, 足以推动任何一位认真的学习者完成并掌握一个实用软件的开发
    • 认真的定义是: 每周主动投入 12小时以上在项目开发/探索/自学/...过程中

期待

既然, ==蠎营-style== 课程真的能帮助大家 got 到如何自学

从具体实施状态中, 可以感受到, 结对绑定后, 的确没了理由可以放松, 面对自己的承诺, 压力空前, 以往零散获得的知识片段, 在规定时间里可以完成什么, 完全靠个人努力, 虽然没有小组成员对比, 但是, 自己和上周自己对比也是可以非常明显感知到变化;

基本吻合最初设想;

如何预约?

  • 通过邮件先报名: askdama@googlegroups.com
  • 将自己想开发的软件以及每周可以投入学员的时间段说明一下
    • 然后, 大妈可以具体为你设计专用路径, 从0.042 开始, 快速进入自己的项目.
